### Phase 11: 동적 신뢰도 시스템

**목표**: 패턴별/사용자별 신뢰도 자동 조정

**작업**:

1. 신뢰도 엔진 (기존 0.75 고정 → 동적)
   ```typescript
   // 기존 (v2.1.0)
   confidence: 0.75  // 모든 패턴 동일

   // 새로운 (v2.2.0)
   confidence: 0.82  // sum (자주 사용)
   confidence: 0.45  // custom_pattern_1 (새로움)
   confidence: 0.71  // filter (때때로 거부됨)
   ```

2. 신뢰도 계산식
   - 기본값: 0.75 (초기)
   - 사용 횟수: +2% (max: 0.95)
   - 승인율: +5% (승인 시)
   - 거부율: -3% (거부 시)
   - 시간 감소: -0.1%/일 (신뢰도 유지 필요)

3. A/B 테스팅
   - 같은 Intent에 다른 패턴 추천
   - 사용자 선택으로 신뢰도 결정
   - 자동 학습

**파일**:
- `src/phase-11/confidence-engine.ts` (400 LOC)
- `src/phase-11/ab-testing.ts` (300 LOC)
- `tests/phase-11-confidence.test.ts` (400 LOC)

**성과**: 신뢰도 정확도 90%+ 달성

### Phase 13: 커스텀 패턴 추가 기능

**목표**: AI가 새로운 패턴 정의 및 추가 가능

**작업**:

1. 패턴 정의 언어 (미니 DSL)
   ```
   @pattern
   name: my_custom_pattern
   input: array<number>, number
   output: number
   description: "사용자 정의 패턴"
   examples:
     - input: [1,2,3], 2 → output: 6
     - input: [10,20], 5 → output: 75
   complexity: O(n)
   ```

2. 패턴 저장소
   - 로컬 저장 (`~/.freelang/patterns`)
   - 클라우드 동기화 (선택)
   - 커뮤니티 패턴 공유

3. 패턴 검증
   - 문법 검사
   - 타입 확인
   - 성능 테스트
   - 안전성 검사

4. CLI 명령어
   ```bash
   freelang --add-pattern my_pattern.free
   freelang --list-custom-patterns
   freelang --remove-pattern my_pattern
   freelang --export-patterns patterns.json
   freelang --import-patterns patterns.json
   ```

**파일**:
- `src/phase-13/pattern-dsl.ts` (600 LOC)
- `src/phase-13/pattern-registry.ts` (400 LOC)
- `src/phase-13/pattern-validator.ts` (300 LOC)
- `tests/phase-13-custom-patterns.test.ts` (500 LOC)

**성과**: AI가 자신의 패턴 정의 가능
